Describe the DOM

The Document Object Model (DOM) is a programming interface for web documents. It represents the page so that programs can change the document structure, style, and content. The DOM represents the document as nodes and objects; that way, programming languages can interact with the page.The DOM is not a programming language, but without it, the JavaScript language wouldn't have any model or notion of web pages, HTML documents, SVG documents, and their component parts.

Difference between Arrays and Objects

Arrays

An array is an arrangement of numbers, pictures or objects formatted into rows and columns according to their type. In coding and programming, an array is a collection of items, or data, stored in contiguous memory locations, also known as database systems. The purpose of an array is to store multiple pieces of data of the same type together

Objects

Objects in JavaScript are core data structures that comprise properties and methods.Objects let you group related data together and split code into logical pieces.

Accessing Data

To access an element from an array, you reference the array name, followed by a pair of square brackets containing the index of the element you want to access. Arrays are zero-indexed, which means the first element in the array has an index of 0, the second element has an index of 1, and so on.

What are Functions

A function is a "subprogram" that can be called by code external Like the program itself, a function is composed of a sequence of statements called the function body. Values can be passed to a function as parameters, and the function will return a value.Defining functions inside other functions in JavaScript allows for the creation of private functions that are only accessible within the scope of the outer function. This can be useful for encapsulating functionality and preventing it from being accessed or modified from outside the outer function. It also helps in organizing and structuring code, making it easier to manage and maintain
